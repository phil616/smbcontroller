卷管理

卷是 Samba 对外发布的共享目录。创建卷会写入 smb.conf；如果勾选自动创建目录，后端会在本机创建对应路径。

共享名路径状态匿名操作

用户管理

SMB 用户会对应 Linux 系统用户，并通过 smbpasswd 设置 Samba 密码。用户名只能包含英文字母、数字和下划线。

用户名类型显示名状态过期时间操作

权限管理

权限矩阵按“用户 × 卷”展示访问规则。选择只读或读写会立即保存并重新生成 Samba 配置；无权限会移除该用户对该卷的访问。

批量授权

超级管理员可一次性给某个用户授权多个卷，或给某个卷授权多个用户。

关于

        

          
SMB Controller 是什么

          
SMB Controller 是一个运行在 Linux 服务器上的 Samba 管理面板，用于通过 Web UI 管理 SMB 共享目录、SMB 用户、访问权限和 Samba 配置生成。

          
它解决什么问题

          
传统 Samba 管理需要手工编辑 smb.conf、维护 smbpasswd、处理 Linux 用户和目录权限。这个项目把这些操作收敛到一个控制面板，并在修改后自动生成配置、修复目录权限、reload smbd。

          
核心能力

          
    
            
  • 创建和管理共享卷，并同步到底层 smb.conf。
    • 
            
  • 创建普通用户和临时用户，临时用户到期后自动清理。
    • 
            
  • 按用户和卷精细化设置无权限、只读、读写权限。
    • 
            
  • 限制可信访问域名，降低管理入口被错误域名访问的风险。
